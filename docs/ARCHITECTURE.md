# VMS Architecture

## Purpose

The Valuation Management System (VMS) is designed as a scalable platform that serves both the public and professionals.

The platform follows a modular architecture so new features can be added without rebuilding the application.

---

# Core Modules

## 1. Authentication

Purpose:
Manage user identity and access.

Planned providers:

- Google
- Microsoft
- Apple (future)
- Yahoo (future)
- Custom Email Login
- Organization Login (Custom Domain)

Guest users may browse public information without signing in.

---

## 2. Map

Purpose:

The map is the heart of VMS.

Features:

- Interactive Nepal Map
- Search by place
- Pin locations
- View pin details
- Layer control
- GPS location
- Future offline support

---

## 3. Property Records

Purpose:

Store and display property information.

Examples:

- Property ID
- Municipality
- Ward
- Location
- Area
- Government valuation
- Attached documents
- Notes

Public records remain viewable.

Editing requires authentication.

---

## 4. Government Information

Purpose:

Provide verified government information.

Examples:

- Government valuation rates
- Survey offices
- Land Revenue Offices
- Municipal contacts
- Official notices

---

## 5. Professional Tools

Initially Free

- Area Converter
- Valuation Calculator

Future VMS Pro

- Advanced valuation
- Professional survey tools
- Record editing
- Premium utilities

---

## 6. User Profile

Features:

- Personal profile
- Saved pins
- Activity
- Subscription status

---

## 7. Administration

Accessible only by administrators.

Responsibilities:

- Verify records
- Moderate public submissions
- Manage government information
- Manage users

---
